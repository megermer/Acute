## Running venv

Install: 
`py -m venv .venv`
> Creates a .venv folder that allows you to create and run venv envs

Create the env:
`python -m venv {env-name}`
> Add this to .gitignore
> So our venv folders with different names won't cause an issue

Activate env: 
`{env-name}\Scripts\activate`

Deactivate env: 
`deactivate`
